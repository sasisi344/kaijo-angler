# マガジン・インテリジェンス・データ蓄積ルール (Magazine Intelligence Data Rules)

このディレクトリは、海上アングラーの「週刊マガジン」および「月刊マガジン」の生成に使用された生データ、解析済みJSON、およびリサーチ結果を蓄積する場所です。
AIが過去のトレンドを学習し、次回の予測精度を向上させるための「知識ベース」として機能します。

## 1. ディレクトリ構造の規則 (YYYY/MM/WW)

データは以下の階層構造で厳格に管理します。

- **`YYYY/`**: 年 (例: `2026`)
- **`YYYY/MM/`**: 月 (例: `2026/03`)
- **`YYYY/MM/WW/`**: 週番 (例: `2026/03/W13`)
- **`YYYY/MM/summary/`**: 月間総括データ (例: `2026/03/summary`)

### 週番 (WW) の定義
- ISO 8601準拠、またはプロジェクト内で定義された週番号（例：3月第4週 = W13）。
- フォルダ内には `research-results.json` および `intelligence-meta.md` を格納する。

## 2. 格納データ形式 (JSON Schema)

各週/月のディレクトリには、以下の基本セットを必ず含めること。

### 1) `research-results.json`
`intel-data-schema.md` に基づいた解析済みデータ。
- キーワード頻出度 (fish_keywords)
- 活性度スコア (activity_score)
- 施設メンション数 (facility_mentions)

### 2) `intelligence-summary.md`
AIによる解析の「思考プロセス」を記録したメモ。
- なぜそのエリアが「Hot」と判定されたか
- 特筆すべきSNS投稿やブログ記事の抜粋
- 次週/来月への予測ロジック

## 3. 月次集積 (Monthly Accumulation)

毎月25日〜末日に、その月の全ての `W**` データを統合し、`summary/` ディレクトリに月間レポート用データを生成・保存する。
この際、週ごとの変動（水温の変化に伴うヒット魚種の遷移など）を「月間トレンド」として正規化する。

## 4. AIインデックス最適化

- **メタデータ**: 各JSONには必ず `timestamp` と `project_id: "kaijo-angler"` を含める。
- **リファレンス**: 記事側で使用した「ヒートマップの重み（weights）」のソースはこのフォルダ内のJSONであること。
- **命名規則**: ファイル名は小文字・ハイフン繋ぎで統一する。
