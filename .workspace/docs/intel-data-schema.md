# 海上アングラー：リサーチ・データセット定義 (Intel Data Schema)

週間・月間マガジンの「ヒートマップ」および「釣果インサイト」を生成するためのデータセット定義です。

## 1. 抽出データセット (Weekly / Monthly)

各エリア（都道府県・海域）ごとに以下の情報を抽出・正規化します。

| プロパティ | データ型 | 説明 | 抽出ソース |
| :--- | :--- | :--- | :--- |
| `area_id` | string | `mie`, `wakayama` などのエリア識別子 | 投稿位置情報/ハッシュタグ |
| `fish_keywords` | map | `{ "マダイ": 45, "ワラサ": 12 }` 魚種別の言及数 | SNS/ブログ本文 |
| `activity_score` | number | 0-100 の活性度。KW数と投稿密度から算出 | 独自アルゴリズム |
| `facility_mentions` | array | `[{"name": "施設A", "count": 5}]` (月間は重要) | SNS/施設ブログ |
| `trend_fish` | string | 今週/今月の「最もアツい」魚種 | AI要約 |
| `is_shore_fishing` | boolean | 釣り堀以外の釣果（地磯・堤防）をフィルタするためのフラグ | AI判定 |

## 2. 週刊マガジン用のデータ構造 (JSON)

```json
{
  "period": {
    "start": "2026-03-24",
    "end": "2026-03-31"
  },
  "area_data": [
    {
      "id": "mie-toba",
      "score": 85,
      "top_fish": "マダイ(放流直後)",
      "volume": 124,
      "facility_links": ["fishing-park-triton"]
    }
  ],
  "hot_keywords": ["サーモン", "シマアジ", "桜鯛"]
}
```

## 3. 2026年2月・3月期 試験運用ロードマップ

1.  **データ収集 (Feb/Mar)**:
    - 過去のSNS（X/Insta）投稿から「海上釣り堀」および「主要魚種」を含む投稿を収集。
    - 施設名が含まれない投稿をどう「地域」へマッピングするかのロジック検証（投稿アプリのGeoTagや、前後の投稿からの推定）。
2.  **スクリプトの効率化**:
    - 全113施設を個別にスクレイピングするのではなく、地域ハッシュタグを優先して「網羅的」に集める手法を開発。
3.  **ヒートマップの正規化**:
    - 投稿が集中する特定施設に引っ張られすぎないよう、地域平均を出す計算式の調整。

## 4. 記事テンプレートへの反映

- **週刊**: KW頻出度をメインに「どの魚を狙うのが効率的か」をアドバイス形式で出力。
- **月刊**: 施設ごとの月間MVPや、最も釣られた魚種をランキング形式で表示。
