# アクセス特化ハブ（travel）— 記事リストと競合対応タスク

**目的**：「◯◯地方・エリアへの最適なアクセス方法」を **検索・回遊のハブ**として独立させ、施設記事は **ラストマイル＋要約**に寄せる。アフィリエイト（交通・宿・レンタカー等）は **ハブ側に集約**する想定。

**既存カテゴリ**：`src/content/blog/column/travel/` に配置。フロントマターの `category` は現状 `釣り旅ガイド` が多い。アクセス専用は **`アクセスガイド`**（または `釣り旅ガイド` のままサブフォルダ `travel/access/` で区別）を定義してから統一する。

## 3. 重複時の方針：新規ハブ優先で既存を「作り直す」

**原則**：地域アクセスの **権威コンテンツはハブ1本（または上表の粒度）に集約**する。既存のモデル記事と重複するなら、**追記修正ではなく構成ごと再設計**する。

- **新規アクセスハブ**：幹線移動・料金比較・お得な切符・レンタカー・宿泊の軸・アフィリエイト枠。
- **既存の `travel` モデルプラン／体験記事（10本）**：**釣行モデルコース・グルメ・観光・施設紹介専用**に再定位し、**幹線アクセスの長文は削除**して新ハブへ誘導（1〜2段落の要約＋内部リンクのみ残す）。`travel/access/` のアクセスガイドは別方針（ハブとの役割分担を定義）。
- **施設記事**：遠方からの複製パラグラフは削除し、**最寄りIC・駅・駐車場・施設固有の注意**＋「遠方からは▶該当ハブ」リンク。

重複が明確な場合は **パッチ当て禁止**とし、**該当記事を全面リライト**するタスクに落とす（下記フェーズB）。
