# 海上アングラー：施設紹介記事 完璧な構成ルール (Point-Rule)

本ドキュメントは、施設紹介記事（fishing-facility）におけるSEOベストプラクティスと、ユーザー体験（UX）を最大化するための標準構成を定義する。

---

## 1. 記事構成（セクション構成）

以下の順番でH2、H3ヘッダーを構成すること。

### 1.1 導入（リード文）
*   **内容**: 施設の特徴を一言で表すキャッチコピー、期待できる体験、解決する悩み。
*   **ルール**: 1文1段落で簡潔に。施設への期待感を高める情緒的な表現を含める。

### 1.2 [施設名]が選ばれる3つの理由（ハイライト）
*   **内容**: H3で3つの独自ポイントを解説。
*   **例**: 「理由1：圧倒的な放流量」「理由2：初心者・女性に優しいホスピタリティ」「理由3：釣った後の地引網体験」など。
*   **目的**: ページを開いて数秒で魅力を伝える（離脱防止）。

### 1.3 施設基本情報と料金システムの完全テーブル化
*   **目的**: ユーザーが知りたいスペック情報を一目で把握できるようにし、情報のスキャン性を最大化する。
*   **基本スペックテーブル**: 以下の項目を網羅したテーブルを作成。
| 項目 | 内容 |
| :--- | :--- |
| **場所・住所** | 〒000-0000 [正確な住所] |
| **営業時間** | [季節による変動も含めて記載] |
| **定休日** | [曜日や祝日の扱い] |
| **電話番号** | [ハイフンあり] |
| **公式サイト** | [リンク形式] |

*   **料金システムテーブル**: コース別の料金体系を明示。
| コース名 | 大人 | 女性・子供 | 利用時間 |
| :--- | :--- | :--- | :--- |
| [例：1日コース] | **00,000円** | 0,000円 | [0:00〜0:00] |
| [例：半日コース] | **0,000円** | 0,000円 | [0:00〜0:00] |

*   **ルール**: 重要な価格や時間は `<strong>` タグを使用し、単位（円）を忘れずに記述すること。

### 1.4 メイン魚種の攻略！おすすめ仕掛けとコツ
*   **タナ（深さ）の戦略**: その施設特有の深さや網の形状に応じた攻略法。
*   **エサのローテーション**: その施設で実績のあるエサの組み合わせ。
*   **コンポーネント**: `<TackleCard id="..." />` を配置し、推奨仕掛けを可視化する。

### 1.5 快適な釣行を支える周辺用品・サービス
*   **内容**: レンタルタックルの有無、エサの販売状況、魚の下処理サービス。
*   **ツール**: クーラーボックスや保冷剤などの紹介。

### 1.6 アクセス情報と周辺ガイド（LTVの向上）
*   **車/電車でのアクセス**: 最寄りICや駅からの具体的な所要時間。
*   **周辺観光**: 釣りの後に寄れる温泉、道の駅、食事処。
*   **目的**: 「釣り×観光」のパッケージとして価値を高める。

### 1.7 【まとめ】[施設名]をおすすめしたい度
*   **筆者判定**: 星評価（★★★★☆）と、ターゲット（「初心者向け」「ガチ勢向け」など）。
*   **最終アドバイス**: 訪問前に必ず準備すべきこと（例：エサの事前購入、氷の持参など）。

## 2. SEO & マークアップ・ベストプラクティス

### 2.1 タイトルタグ（Frontmatter）
*   **形式**: `【都道府県名】施設名｜[特徴キーワード]・料金・アクセス情報`
*   **ルール**: 重要なキーワードを左側に寄せる。

### 2.2 強調表現 (`<strong>`)
*   **ルール**: 読み飛ばし（スキャン）しても内容が理解できるよう、重要な単語や数字（料金、タナ、魚種、IC名）を必ず `<strong>` タグで囲む。

### 2.3 フロントマターの整合性
*   `google_maps` と `facility_details` のデータは、本文の内容と100%一致させること。AIが本文からデータを抽出する際の「正解」として機能させる。

### 2.4 画像の最適化
*   `cover.jpg` のALTテキストには「[施設名]でマダイを釣り上げる様子」など、SEOを意識した説明を付与する（フロントマターにはないが、生成時のプロンプトやHTML変換を意識）。

- [ ] 料金体系は表形式で一目でわかるか？
- [ ] その施設特有の「釣るためのコツ」が具体的に書かれているか？
- [ ] `.data-set` にある科学的根拠（水温、視覚特性など）を引用しているか？
- [ ] 釣り以外の楽しみ（周辺観光）を提案できているか？
